# Qeetrix

**Qeetrix** is the shared design system for **Qeet Group** products. It is built on
**Tailwind CSS v4** and **shadcn/ui with the Base UI registry**, and themed by a single,
token-driven source of truth.

## The package

Everything ships in **one** package — **`@qeetrix/ui`**. Install it and you get components,
brand, and tokens with nothing else to wire up:

- **Components** — the React component library (this workshop), at `@qeetrix/ui`.
- **Tokens** — design tokens (W3C JSON → Style Dictionary → CSS + JSON) generate the
  `:root` / `.dark` runtime variables (baked into `@qeetrix/ui/styles.css`), plus a
  `--qx-`-prefixed raw export at `@qeetrix/ui/tokens.css` and resolved JSON at
  `@qeetrix/ui/tokens.json`.
- **Brand** — the Qeet logo (theme-adaptive) and custom icons, at `@qeetrix/ui/brand`.

## Using it

```tsx
import { ThemeProvider, Button } from "@qeetrix/ui";
import "@qeetrix/ui/styles.css";

<ThemeProvider defaultTheme="system">
  <Button>Authenticate with Qeet</Button>
</ThemeProvider>;
```

## Theming

Light and dark are first-class and driven by the `.dark` class on the root element — flip the
**Theme** toolbar above to preview either. Colours are currently a neutral/greyscale ramp; the
brand palette is a documented open decision (OD-DS-03) and drops in by editing only the primitive
ramps in the token source (`packages/ui/tokens`).

See **Foundations → Colors / Typography / Spacing & Radius** for the token galleries, and
**Primitives** / **Brand** for components.
