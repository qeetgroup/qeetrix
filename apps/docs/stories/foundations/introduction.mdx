## One package

Everything ships in **`@qeetrix/ui`** — components, brand and tokens. Install it and there is nothing
else to wire up:

```bash
pnpm add @qeetrix/ui
```

```tsx
import { ThemeProvider, Button } from "@qeetrix/ui";
import "@qeetrix/ui/styles.css";

export function App() {
  return (
    <ThemeProvider defaultTheme="system">
      <Button>Authenticate with Qeet</Button>
    </ThemeProvider>
  );
}
```

What's in the box:

- **Components** — the React library you're browsing in this workshop, at `@qeetrix/ui`.
- **Tokens** — design tokens authored as W3C JSON, compiled by Style Dictionary into the `:root` / `.dark`
  runtime variables baked into `@qeetrix/ui/styles.css`, plus a raw `--qx-`-prefixed export at
  `@qeetrix/ui/tokens.css` and resolved JSON at `@qeetrix/ui/tokens.json`.
- **Brand** — the theme-adaptive Qeet logo and product icons, at `@qeetrix/ui/brand`.

## Theming

Light and dark are first-class, driven by the `.dark` class on the root element — the same strategy the
Qeetrix `ThemeProvider` uses in production, so this workshop renders identically to your app. Colours are
currently a neutral / greyscale ramp; the Qeet brand palette is a documented open decision (`OD-DS-03`)
and drops in by editing only the primitive ramps in the token source (`packages/ui/tokens`).
