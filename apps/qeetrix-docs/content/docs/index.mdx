Qeetrix docs now track the source in `@qeetrix/react`, not the default
starter template. The current workspace now documents both the React package
and the shared `@qeetrix/icons` icon library: semantic tokens, theme
orchestration, primitives, utilities, and a generated icon set.

<Callout title="Current scope">
  The package currently exports `ThemeProvider`, `useTheme`, `Button`,
  `buttonVariants`, `Accordion`, `AccordionItem`, `AccordionTrigger`,
  `AccordionContent`, `cn`, and the generated `@qeetrix/icons` library with base
  icon components and style variants.
</Callout>

## Package priorities

- Keep semantic tokens consistent across light, dark, and system themes.
- Ship composable React primitives built on top of Base UI.
- Provide a searchable icon system with consistent variant naming and copyable usage.
- Stay aligned with Tailwind 4 and shared utility-based styling.
- Keep the docs grounded in what exists in source today.

## Current surface area

| Area        | What ships now                                                                                                      |
| ----------- | ------------------------------------------------------------------------------------------------------------------- |
| Foundations | Semantic color tokens, radius scale, typography defaults, dark-mode class strategy                                  |
| Theming     | `ThemeProvider`, `useTheme`, local storage persistence, system preference sync, cross-tab updates                   |
| Components  | `Button`, `Accordion`, `AccordionItem`, `AccordionTrigger`, `AccordionContent`                                      |
| Icons       | Base icon exports, variant switching (`bold`, `broken`, `bulk`, `linear`, `outline`, `twotone`), generated registry |
| Utilities   | `buttonVariants`, `cn`                                                                                              |
